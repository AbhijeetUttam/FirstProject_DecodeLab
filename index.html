Key Concepts

The engineering principles behind this project

IA

Information Architecture

Structure before code. Organize content for findability. Sitemap → Navigation. Minimize cognitive load.

</>

Semantic HTML

Use <header>, <main>, <article>, <footer>. No "div soup". One <h1> per page.

CSS

CSS Engineering

No inline styles. External files only. DRY principle via BEM. One master.css for all pages.

⊞

Layout Systems

CSS Grid for 2D macro structure. Flexbox for 1D component alignment. Right tool for the right job.

A11Y

Accessibility

Alt text on every image. Contrast ratio > 4.5:1. Accessible labels. The web must be perceivable by all.

QA

Quality Gate

W3C validation — zero errors. Semantic audit. Visual regression — pixel-perfect match to design.
